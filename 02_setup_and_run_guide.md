## Requirements

### Python packages

```bash
pip install pyspark torch pandas numpy fastapi uvicorn pyarrow
```

### Hadoop (Windows only — required for PySpark)

1. Download winutils from: https://github.com/steveloughran/winutils
2. Extract to `C:\hadoop\bin\`
3. The notebooks already set the path automatically:

```python
os.environ["HADOOP_HOME"] = "C:\\hadoop"
os.environ["PATH"] += ";" + "C:\\hadoop\\bin"
```

### GPU (optional but recommended for training)

Install PyTorch with CUDA support from: https://pytorch.org/get-started/locally/

The training notebook auto-detects GPU:
```python
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
```

## Step 4 — Run the API

Make sure these 3 files are in the same directory as your project root:
- `drought_gru_model.pth`
- `scaler.pkl`

Then navigate to the `api/` folder and start the server:

```bash
cd api
uvicorn main:app --reload
```

**Expected output:**
```
Initializing system resources... Loading PyTorch model.
PyTorch weights successfully loaded into memory state.
INFO:     Uvicorn running on http://127.0.0.1:8000
```

If you see a CRITICAL error about the model path, check that `drought_gru_model.pth` and `scaler.pkl` are one folder above `api/`. The paths in `main.py` are set to `../`:

```python
MODEL_PATH  = "../drought_gru_model.pth"
SCALER_PATH = "../scaler.pkl"
```

## Feature Input Ranges (Reference)

Use these ranges when entering real weather data:

| Feature | Unit | Typical Range |
|---|---|---|
| PS | hPa | 95 – 105 |
| elevation | meters | 0 – 4000 |
| T2M_MAX | °C | -30 – 50 |
| QV2M | g/kg | 0 – 30 |
| WS10M_RANGE | m/s | 0 – 20 |
| WS50M_RANGE | m/s | 0 – 25 |
| NVG_LAND | % | 0 – 100 |
| GRS_LAND | % | 0 – 100 |
| FOR_LAND | % | 0 – 100 |
| CULTRF_LAND | % | 0 – 100 |
| CULTIR_LAND | % | 0 – 100 |
| CULT_LAND | % | 0 – 100 |
| lat | degrees | -90 – 90 |
| lon | degrees | -180 – 180 |
| T2MDEW | °C | -40 – 35 |
| T2MWET | °C | -30 – 40 |
| T2M_RANGE | °C | 0 – 40 |
| TS | °C | -40 – 60 |

> Important: Send raw real-world values. The API applies scaling internally before passing to the model.

## Common Errors

| Error | Cause | Fix |
|---|---|---|
| `CRITICAL: System could not locate weights` | `.pth` or `.pkl` not found | Check paths in `main.py` — files must be one level above `api/` |
| `Matrix error: Must contain exactly 7 days` | Sent less or more than 7 day objects | Always send exactly 7 dicts inside `data` |
| `Model or scaler not loaded` | Startup failed silently | Check terminal logs for the real error |
| `TypeError: cannot pickle '_thread.RLock'` | Tried to pickle PySpark scaler directly | Save scaler as `{"mean": ..., "std": ...}` dict instead |
| Predictions seem wrong | Forgot to apply scaling | Confirm `scaler.pkl` exists and `main.py` loads it |
